Page 1 · Impact sizingProcess & Organization Readiness

Before investing in agents, answer two questions — "can we verify the output?" and "is the organization ready?"

PRS and ORS score each workflow on evidence before design spend is committed · Score 0.0–2.0 per axis · threshold 1.2 splits High/Low · PRS × ORS → Page 2

PRS Process Readiness

"Can we define 'correct' — and verify without an expert every time?"

Output structure
Fixed schema vs. free text — BOM line items and PO qty fields score high; NPI design notes score low.
Correctness verifiability
System pass/fail vs. expert judgment — component qty vs. MRP is rule-checkable; design intent needs a human eye.
Variance tolerance
Multiple valid answers OK — alternate approved suppliers fine; safety-critical spec deviations are not.
Ground truth availability
Historically approved examples exist — years of matched PO records score high; new prototype workflows score low.
HighSupplier PO 3-way match — ERP qty ties to GRN; 50k+ records; pass/fail is binary.
LowOEM co-design brief review — "meets industrial-design intent?" — subjective; no agreement.
ORS Organization Readiness

"Is the organization ready to adopt and sustain this change?"

Sponsor authority
Mandating exec with budget — plant VP with headcount authority scores high; nominal coordinator scores low.
Team receptivity
Will frontline teams use it — line leaders requesting SMT triage automation score high; fear of headcount cuts scores low.
Integration complexity
Systems landscape cleanliness — one MES + SAP with APIs scores high; five legacy systems and spreadsheets score low.
Change track record
Prior automation success — recent RPA win on IQC scoring scores high; stalled MES rollout scores low.
HighSMT yield exception routing — VP Ops sponsored, MES APIs live, line leaders requesting automation.
LowCross-site component reallocation — six legacy systems, no single owner, prior MES project stalled.

How we size impact
① Business Value — "If we win, what's it worth?"
Cost savings
Labor · errors · penalties
FTE hours removed, rework cost avoided, supplier penalty exposure reduced.
Quality uplift
Defect · rework · scrap
Reduction in mis-picks, wrong-component installs, and IQC escapes.
Speed gain
Cycle · queue · lead time
Hours shaved from NPI sourcing, RFQ turnaround, or shortage-allocation queue.
Strategic lift
CEO · CFO · board narrative
How visibly the workflow advances a stated corporate priority.
② Agentic Fit — "Is an agent the right tool?"
Binary routing
Yes/no · single rule
"If lead time > 14 days → escalate to commodity manager" — simple condition; RPA may suffice.
Multi-path choice
Several options · factors
"Based on customer tier, margin, inventory → pick from 5 allocation options" — agent earns its keep.
Judgment call
Trade-offs · context
"Evaluate supplier quote across cost, quality, geopolitical risk → recommend action" — high agentic fit.

Rule of thumb · Mostly execution → RPA likely cheaper · Decision-heavy → agent is the right investment.

Page 4 · DesignFive agent archetypes

Classify every workflow step as one of five agent archetypes — architecture becomes a classification exercise

Mutually exclusive · collectively exhaustive · each type has a clear selection trigger

Orchestrator

Coordinator
Core function
  • Decomposes work into sub-tasks
  • Routes tasks to agents
  • Maintains workflow state
When to use
  • Step coordinates multiple sub-tasks or agents
Tools wired
  • Task queue · routing rules · state store
e.g. NPI intake router — triages cases to specialist agents

Executor

Doer
Core function
  • Performs defined operations
  • Calls APIs and system actions
  • Writes records
When to use
  • Well-defined action with clear I/O
Tools wired
  • ERP/MES APIs · messaging · file writers
e.g. Invoice posting agent — matches PO/GR and posts to ledger

Analyst

Reasoner
Core function
  • Synthesizes multiple inputs
  • Makes recommendations
  • Compares against criteria
When to use
  • Reasoning or trade-offs over inputs
Tools wired
  • Calculators · scoring models · LLM reasoning
e.g. Component shortage analyst — recommends allocation across plants

Retriever

Knowledge gatherer
Core function
  • Finds relevant context
  • Surfaces evidence
  • Ranks by relevance
When to use
  • Step needs to find information first
Tools wired
  • RAG over docs · vector search · SQL
e.g. Approved vendor list lookup — finds applicable sourcing rules

Evaluator

Guardrail
Core function
  • Validates agent outputs
  • Enforces quality & compliance
  • Flags exceptions
When to use
  • Quality or compliance judgment required
Tools wired
  • Rule engines · LLM-as-judge · HITL hooks
e.g. Export-control reviewer — validates disposition before closure
Classify · Need info → Retriever · Need reasoning → Analyst · Need validation → Evaluator · Coordinate → Orchestrator · Else → Executor.  |  Start simple · Test one Executor + tools before composing multiple types.

Page 5 · DesignThree Agent–Human interaction modes

Assign each step a mode — Autopilot · Co-Pilot · Guardian — based on worst plausible failure severity

Trust contract between agent and employee · selection is risk-based, not preference-based

Autopilot

Full autonomy — execute and log

When to choose

  • Low-risk, high-volume, reversible
  • Verifiable by post-hoc sampling
  • Recovery cost < review cost

Design implications

  • Comprehensive audit trail
  • 5–10% post-hoc sampling
  • Anomaly detection → auto-escalation
e.g. PO 3-way match within tolerance — auto-post, sample 5% weekly
Co-Pilot

Shared — agent proposes, human refines

When to choose

  • Medium-risk, judgment-dependent
  • Benefits from human refinement
  • Inline review volume manageable

Design implications

  • Draft + diff UI for reviewer
  • Feedback loop captures edits
  • Time-out → escalate or revert
e.g. Shortage allocation draft — planner edits before ERP update
Guardian

Limited — agent prepares, human decides

When to choose

  • High-risk or irreversible decisions
  • Regulated or audit-critical outputs
  • Error cost >> review cost

Design implications

  • Agent assembles evidence, never acts
  • Decision UI with explainability
  • Human signature on every case
e.g. Export-control disposition — compliance officer approves every case
Selection rule · "If the agent gets this wrong, what's the worst outcome?" — Reversible/cheap → Autopilot · Reworkable → Co-Pilot · Irreversible/regulated → Guardian.
