Practitioner runbook

Agentic workflow delivery field guide.

A facilitation-ready deck for consultants running the playbook: what to ask, what to score, what to build, and when to stop.

How to use this deck

  • Run one phase at a time.
  • Do not skip gates to satisfy timeline pressure.
  • Record evidence as you go.
  • Translate every decision into a deliverable artifact.

Delivery route

The field guide runs as four workblocks with explicit artifacts.

Week 0

Size

Screen candidates, classify risk, rank the portfolio.

Artifact: priority brief.

Days 1-10

Design

Map workflow, select agent patterns, define acceptance criteria.

Artifact: architecture pack.

Weeks 3-6

Implement

Build thin slice, run evals, shadow real work.

Artifact: MVP evidence report.

Months 2-4

Scale

Deploy production controls and plan next waves.

Artifact: operating model.

Facilitator rule

If the team cannot name the evidence behind a decision, the decision is not ready. Pause and collect the missing fact before moving forward.

Run the sizing workshop

Day 1 eliminates structurally unready workflows before scoring begins.

30-minute interview script

0-5 min

Ask the owner to walk from trigger to final output.

Listen for undocumented paths.

5-20 min

Probe the six binary readiness criteria.

Ask for evidence, not opinions.

20-25 min

Ask what is changing in the next six months.

Check process stability.

25-30 min

Confirm sponsor authority and decision speed.

Name the owner.

Pass rule

Require 5 of 6 Yes answers. One exception is allowed only with a mitigation plan, owner, and impact on timeline.

Process can be documented or described in under 2 hours.

Input data is digitally accessible.

Process runs more than 50 times per month or has justified high-value exception.

Willing process owner is identified.

Good output can be defined or shown.

Process is stable for six months.

ODS threshold

High if score is at least 1.2. Tests whether output correctness can be objectively verified.

ORS threshold

High if score is at least 1.2. Tests sponsorship, receptivity, integration complexity, and change history.

Priority threshold

Proceed to Design when PriorityScore is at least 3.0 and sponsor accepts the action path.

Run the design sprint

Use the 10-day sprint to convert the selected workflow into a buildable architecture.

Days 1-3

Discover

Map trigger, inputs, steps, decisions, exceptions, outputs, and current controls.

Sync 1: complexity decision.

Days 4-7

Design

Select agent archetypes, interaction modes, tools, data, and handoff contracts.

Sync 2: architecture v1.

Days 8-10

Validate

Walk through scenarios, acceptance tests, risks, and MVP scope.

Sync 3: ready for MVP gate.

Complexity gate

Ask first: can the workflow be handled by one agent with tools? Use multi-agent design only when distinct capabilities, ownership boundaries, or validation controls require it.

Autopilot

Low-risk, reversible, high-volume, measurable correctness.

Co-Pilot

Medium-risk, judgment-dependent, human refines agent output.

Guardian

High-risk, irreversible, regulated, or subjective. Human decides.

Run MVP validation

The MVP phase proves repeatability through testable evidence.

Build sequence

Implement one end-to-end thin slice.

Connect required tools, data sources, and permissions.

Add evaluator checks and decision logging.

Run golden examples and exception cases.

Run in shadow mode with real users.

Evidence to collect

  • Output quality by case type.
  • Cycle time versus current process.
  • Exception frequency and escalation quality.
  • Integration failures and recovery behavior.
  • User acceptance signals and trust blockers.

Stop condition

If quality cannot be measured, do not proceed to production. Return to Design and add evaluators, better examples, or a Guardian interaction pattern.

Production gate checklist

Use this gate before sponsor go-live approval.

Technical readiness

Integrations work under realistic conditions.

Retries, fallbacks, and dependency failures are defined.

Access controls and audit logs are validated.

Monitoring and support ownership are assigned.

Operating readiness

Process owners know when to accept, override, or escalate.

Shadow users show adoption intent.

Business case signals justify run cost and controls.

Sponsor approves production scope and rollout path.

Scale-out playbook

After the lighthouse workflow, scale by portfolio wave.

Wave 1

Quick Wins

High ODS and high ORS. Prove value in months 1-3.

Wave 2a

Invest & Prove

Low ODS and high ORS. Build human evaluation infrastructure.

Wave 2b

Sponsor & Align

High ODS and low ORS. Warm the organization before deployment.

Wave 3

Defer & Mature

Low ODS and low ORS. Reassess after maturity gaps close.

Reusable asset capture

After each workflow, extract reusable agent specs, prompts, evals, connector patterns, risk decisions, training notes, and support learnings. This is what turns projects into a scalable methodology.

Engagement closeout

Close every engagement with a capability transfer, not only a production launch.

Client handoff

  • Runbook and support owners.
  • Escalation paths and SLAs.
  • Known limitations and roadmap.

Delivery team handoff

  • Reusable assets added to library.
  • Gate decisions archived.
  • Lessons learned documented.

Next-wave decision

Use the portfolio created in Week 0 to select the next workflow, with updated evidence from the lighthouse deployment.

Definition of done

The workflow is live, monitored, owned, explainable, and valuable, and the organization has a clear path to repeat the method on the next workflow.
