Design decision tree

Start with the lowest viable complexity and escalate only when the workflow demands it.

Q1. Can one agent plus tools complete the workflow?

If yes, use a single-agent design. Add evaluator and HITL controls only where risk or correctness requires them.

Single agent with tools

Q2. Do steps require distinct capabilities or owners?

If yes, separate into agent roles using the five archetypes and document handoff contracts.

Multi-agent architecture

Q3. Can output correctness be automatically checked?

If yes, use evaluator loops and automated acceptance tests. If no, design Guardian or Co-Pilot review.

Evaluator or HITL validation

Q4. Is deployment adoption-ready?

If no, scope trust-building, shadow mode, role redesign, and sponsor alignment into production planning.

Agent system components

The five archetypes make architecture decisions teachable and auditable.

Decomposes work, routes tasks, maintains state.

02

Performs defined operations through tools or APIs.

03

Analyst

Synthesizes inputs and makes recommendations.

04

Retriever

Finds and supplies context from knowledge sources.

05

Evaluator

Checks quality, safety, compliance, and completeness.

Architecture ruleWhy it matters
Every agent has one primary archetype.Prevents vague agents that retrieve, reason, execute, and evaluate without clear accountability.
Every agent has typed inputs and outputs.Creates handoff contracts that engineers can implement and validators can test.
Every high-risk output has an evaluator or human checkpoint.Turns trust into an explicit control design instead of a launch assumption.

Human-agent contracts

Interaction mode is selected by risk, reversibility, and validation confidence.

Autopilot

Execute and log

Low-risk, reversible, high-volume tasks where correctness is measurable.

Example: structured invoice matching.

Co-Pilot

Propose and refine

Medium-risk work where the agent drafts or recommends and humans tune the result.

Example: customer response drafting.

Guardian

Prepare and escalate

High-risk, irreversible, regulated, or subjective decisions where humans decide.

Example: compliance judgment.

Blueprint principle

Do not treat human review as a generic fallback. Specify the trigger, reviewer role, expected decision, SLA, and what the agent does after the decision.

Build and prove

The MVP is an evidence factory, not a demo factory.

Thin slice

Build one complete path from trigger to output, including logging and escalation.

Golden set

Use historical approved outputs, explicit rules, or human evaluation panels.

Exception bank

Test common failure modes, boundary cases, integration outages, and ambiguous inputs.

Decision log

Capture why the agent acted, what evidence it used, and where humans intervened.

Production and portfolio

Scale-out adds operational disciplines around the agent system.

Production controls

Run operations

operating layer

Reusable assets

Portfolio waves

Quick Win first, then Invest & Prove and Sponsor & Align as trust and validation maturity increase.

Scale condition

Only scale when the lighthouse workflow has measurable production value, a stable support model, and reusable delivery assets that reduce the cost and risk of the next workflow.
